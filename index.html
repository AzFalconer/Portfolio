Bio ~

Hi, my name is David Darby. I'm a world traveling, bookstore owning, falconer, who is really good at bar trivia. And I code for fun.

I started coding when I got my Atari 800XL in 1984. I mowed lawns all summer to raise the money for it. The good old days when you could write a whole game in just 16K chunks, now that was real programming. My first and only paid coding job (since then I've just coded for friends, family or local non-profits/small business...) was dBase III. Boy that was a long time ago...

I would like to get a job coding, but with no formal education or certifications, I decided to give freeCodeCamp a try. I'm still getting used to using codepen, as I prefer to write html using notepad or in production using nano (ready for the vim flamers...3...2...1) I mostly work in the production environment, working off line is for the weak. :) And did I mention there is a thin line between bravery and stupidity? Really I only work in production on my own sites/apps, when working for someone else I work offline.

Application Programming - In my bookstore, I have written (and continuously upgrade) two programs: a POS (Point of Sale) and an Inventory Control app. Both are written in VB.net using Visual Studio (Don't be a hater, VB works fine for Window's Form Apps.) Both programs utilize a thermal receipt printer and a bar code scanner. Also I have a little VB app that uses a webcam to take a series of 3 pics (1 sec apart) when someone uses a credit card. Never had a cc complaint, but I have pics if I ever do...

Web site design - I have created a few websites over the years for myself and others. Currently I only manage one website myPaperBacks.com for the bookstore. Oh and there is a mobile version also. I really believe in efficiency so I keep my html pretty lean. Give gtmetrix a try if you want to test your own website(s). I utilize php for server side scripting. If I can't score in the high 90% I consider that a failure. Since I taught my self html, mostly looking at the source code for websites I liked, I'm sure I've developed some bad habits. For example I'm trying really hard not to use tables for layout in this course... But tables are so much easier! (Yes, I know it is bad form...)

Server - I run my own e-mail server and webserver on digital ocean droplets. Both servers run Ubuntu 14.04, and the webserver is Apache. (FYI, that was bad security hygiene... The web doesn't need to know where you host and/or what OS & software you run.) Most people advise against running your own e-mail server, but I enjoy it. Also I stood the e-mail server up in Toronto... Maybe I'm just paranoid. And during the Olympics I was able to watch the Canadian coverage utilizing VPN through that server. Having a server in a foreign country has lots of advantages.

Hardware - Hardware was my first love. I've tinkered with electronics as long as I can remember, I got my AS in Electronic Technology. While going to college for an EE degree, I decided I had had enough of theory and was ready to start building things. So I dropped out and joined a small company, Cisco Systems... Maybe you have heard of them?

Future Projects - I'd like to play around with freeBSD and maybe setup a web server using nginx. Also I want to setup the bookstore inventory app to use MySQL.

Free Code Camp - I heard about Free Code Camp while listening to a DTNS podcast. Sounded like a great idea.

Portfolio ~

Website for Retail bookstore. Has ~25 static pages, and takes up ~9.5Mb on the webserver. Since I reset the counter in 2014 it has gotten 2.2 million page views. The main page has a GTmetrix score of 92% and takes an average load time of 413ms. It utilizes php for server side scripting, builds pages dynamically for displaying search results, date specific info, etc...
